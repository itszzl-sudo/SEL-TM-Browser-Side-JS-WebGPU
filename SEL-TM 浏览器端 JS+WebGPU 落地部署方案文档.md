# SEL\-TM 浏览器端 JS\+WebGPU 落地部署方案文档

## 1\. 方案概述

本方案基于自进化受限布局图灵机（SEL\-TM）核心理论，实现**浏览器原生无内核改造、纯前端架构**的布局渲染体系。彻底脱离浏览器原生 WebKit、Blink 等内置布局渲染引擎，采用「JS 冷路径自进化 \+ WebGPU 热路径高速渲染」的异构分层架构。

依托专属宿主页面\+全屏 WebGPU 画布的设计，实现宿主页面零布局占用、外部自定义 HTML 资源加载、SEL\-TM 规则解析、GPU 并行布局计算与画面渲染的全链路自研闭环，是适配浏览器环境的轻量化、可进化、高性能前端布局渲染落地方案。

## 2\. 整体架构设计

### 2\.1 核心架构分层

严格遵循 SEL\-TM 冷热路径分离理论，结合浏览器原生能力做工程化落地，分层职责清晰、无逻辑冗余、完全匹配理论模型：

- **冷路径层（JavaScript 实现）**：承载 SEL\-TM 所有自进化核心逻辑，低频串行执行，负责外部 HTML 加载、DOM 结构解析、CSS 规则识别、能力缺失检测、自修补算子执行、状态机（Q/δ）动态扩展、LMDB 记忆数据读写与固化，不参与任何数值渲染计算。

- **热路径层（WebGPU 实现）**：承载所有高频布局数值计算与画面渲染，并行执行，负责盒模型计算、Flex/Grid/绝对定位等布局求解、像素坐标映射、批量页面渲染，无复杂分支逻辑，最大化发挥 GPU 并行算力优势。

- **载体层（专属 index\.html 宿主）**：极简空白宿主页面，零布局占用，仅提供全屏 WebGPU 画布容器，屏蔽浏览器原生渲染流，独占视口所有显示空间。

- **记忆持久化层（LMDB）**：沿用 SEL\-TM 纯 LMDB 极简架构，存储 W3C 规范长期记忆库（𝕂）、运行习得短期记忆集（𝕃），实现浏览器重启、页面刷新后进化能力不丢失。

### 2\.2 核心特性与落地优势

- 宿主零侵入：index\.html 无任何布局占位、无滚动条、无冗余节点，完全释放视口空间给 WebGPU 画布

- 原生无依赖：基于浏览器标准 WebGPU、Fetch API 实现，无需编译内核、无需服务端部署、无第三方引擎依赖

- 理论强匹配：严格贴合 SEL\-TM 非图灵完备、可自进化、冷热双路径核心特性，无玄学优化、无逻辑篡改

- 高性能渲染：布局数值计算全并行化，热路径剔除浏览器原生渲染冗余，吞吐与帧率远超传统浏览器渲染流程

- 可迭代进化：JS 冷路径支持运行时动态补齐 W3C 新规范特性，无需升级浏览器内核

## 3\. 宿主页面（index\.html）规范定义

### 3\.1 页面布局约束（强制规范）

宿主页面作为纯粹的运行容器，必须满足零布局占用要求，具体约束如下：

- html、body 标签全屏覆盖，外边距、内边距、边框全部清零，盒模型统一标准化

- 禁用页面滚动、默认背景、原生渲染冗余样式，杜绝任何原生布局干扰

- 页面唯一 DOM 元素为 WebGPU 专属画布，画布宽高 100% 自适应视口，无拉伸、无留白、无偏移

- 所有业务 HTML、CSS 资源均为外部加载，不写入宿主页面，彻底隔离容器与业务渲染逻辑

### 3\.2 宿主核心能力

- WebGPU 环境初始化、设备适配、渲染管线创建与管理

- 外部 HTML 资源异步加载、文本资源托管

- JS 冷路径自进化逻辑初始化、记忆系统挂载

- 冷热路径数据通信、任务分发、渲染循环调度

## 4\. 全链路运行流程

### 4\.1 启动初始化流程

1. 浏览器加载 index\.html，初始化极简页面容器，生成全屏 WebGPU 画布

2. 检测浏览器 WebGPU 兼容性，完成 GPU 适配器、设备、上下文配置

3. 加载 LMDB 持久化记忆数据，回填 SEL\-TM 已习得布局能力，完成机器状态初始化

4. JS 冷路径启动，初始化规范匹配、能力检测、自修补算子核心逻辑

### 4\.2 业务资源加载流程

1. 宿主通过 Fetch API 异步加载外部业务 HTML 文件资源

2. JS 冷路径接收 HTML 源码，完成 DOM 结构解析、CSS 样式提取、布局特征扫描

3. 匹配 LMDB 短期记忆集（𝕃），判断当前布局能力是否已习得

### 4\.3 冷路径自进化流程（低频首次执行）

1. 检测到未知布局特性、缺失布局计算能力，触发 SEL\-TM 自修补算子

2. 查询 LMDB 中长期记忆库（𝕂）结构化 W3C 规范规则

3. 动态扩展机器状态集合 Q、改写状态转移函数 δ，生成全新布局执行链路

4. 将新增布局能力、状态逻辑固化写入短期记忆集，完成运行时进化

5. 生成标准化、可并行的布局计算任务，下发至 WebGPU 热路径

### 4\.4 热路径 GPU 渲染流程（高频循环执行）

1. WebGPU 接收 JS 下发的标准化布局任务，基于固化状态链路执行纯数值并行计算

2. 批量求解 DOM 元素尺寸、坐标、层叠、排版等布局数据

3. 通过 GPU 渲染管线完成像素绘制、画面合成

4. 将最终画面输出至全屏画布，持续执行渲染循环

## 5\. 冷热路径详细设计

### 5\.1 JS 冷路径设计规范

冷路径全程由 JavaScript 实现，专注动态逻辑处理，不参与任何 GPU 计算，核心能力包含：

- **资源解析能力**：解析外部 HTML 结构、CSS 布局规则、样式优先级、文档流特性

- **能力检测能力**：对比当前机器习得能力与业务布局需求，精准识别能力缺口

- **自进化能力**：基于 W3C 规范动态新增布局状态、更新转移逻辑、绑定标准算法

- **记忆管理能力**：读写 LMDB 数据库，管理长期规范库与短期习得缓存，保证进化能力持久化

- **任务调度能力**：将结构化布局任务格式化，适配 WebGPU 并行计算格式，实现冷热路径解耦

特性：串行执行、低频触发、多分支动态逻辑、无性能瓶颈，完全规避 GPU 架构短板。

### 5\.2 WebGPU 热路径设计规范

热路径全程由 WebGPU 实现，专注高频数值并行计算，所有逻辑为确定性无分支运算：

- **布局计算**：实现盒模型、常规文档流、Flex、Grid、浮动、绝对定位等标准布局并行求解

- **像素渲染**：完成颜色填充、矩形绘制、图层合成、视口适配等画面渲染能力

- **并行调度**：单 DOM 元素布局计算绑定单 GPU 线程，实现海量元素同时排版计算

- **零拷贝渲染**：依托浏览器统一内存机制，实现 JS 与 GPU 数据无中转传输

特性：并行执行、高频循环、纯数值运算、无动态逻辑，最大化释放 GPU 算力优势。

## 6\. 记忆持久化架构（纯 LMDB）

原 SEL\-TM 工业级架构采用纯 LMDB 持久化方案，但**标准浏览器网页环境原生不支持 LMDB**（LMDB 为本地磁盘数据库，依赖 Node\.js 本地文件系统权限、进程内存映射能力，浏览器沙箱机制严格隔离本地磁盘读写，禁止网页直接操作本地 LMDB 数据库）。针对纯浏览器网页运行场景，本节明确 LMDB 可用性边界，并提供完全兼容 SEL\-TM 双层记忆架构、可无缝替代的浏览器原生持久化方案，全程无服务端、无本地权限依赖。

### 6\.1 浏览器无法直接使用 LMDB 的核心原因

- **沙箱安全限制**：浏览器网页运行在隔离沙箱中，禁止直接读写本地磁盘文件，而 LMDB 基于磁盘文件映射（mmap）运行，无磁盘访问权限则无法启动。

- **运行环境依赖**：原生 LMDB 依赖操作系统进程、本地文件句柄、底层内存映射能力，仅支持 Node\.js、客户端桌面程序、服务端环境，不适配纯前端网页。

- **权限合规限制**：网页程序无本地数据库创建、读写、持久化落地权限，直接引入 LMDB 会触发浏览器安全拦截，无法正常运行。

### 6\.2 最优替代方案：IndexedDB（浏览器原生数据库）

**IndexedDB 是纯浏览器网页下 LMDB 的唯一等效替代**，完全适配 SEL\-TM 记忆架构，可1:1复刻长期记忆𝕂、短期记忆𝕃的存储能力，具备持久化、大容量、异步读写、重启不丢失的核心特性，是W3C标准浏览器API，无兼容性门槛、无安全风险。

#### 6\.2\.1 核心适配能力（对标 LMDB）

- **持久化存储**：页面刷新、浏览器重启、进程崩溃后，数据永久保留，彻底解决重复“学习”问题。

- **大容量存储**：远超 LocalStorage 存储上限，可完整存储结构化 W3C 规范库、机器状态集Q、转移函数δ、习得布局能力缓存。

- **异步无阻塞**：读写操作异步执行，不阻塞JS冷路径逻辑与WebGPU渲染循环，不影响页面性能。

- **结构化存储**：支持存储对象、数组、结构化规则数据，完美适配 SEL\-TM 机器可执行规则格式。

- **原生安全合规**：浏览器原生支持，无需额外权限，完全规避沙箱拦截问题。

#### 6\.2\.2 SEL\-TM 记忆架构适配改造

完全兼容原有冷热路径、自修补算子逻辑，无功能阉割，仅替换存储介质：

- **长期记忆库（𝕂）**：将结构化 W3C HTML/CSS 标准规则集、补丁算法规范、布局状态定义，预加载固化至 IndexedDB 只读仓库，对标 LMDB 永久标准库能力。

- **短期记忆集（𝕃）**：JS冷路径自进化生成的新增状态、改写转移逻辑、固化布局能力，动态写入 IndexedDB，替代 LMDB 习得缓存。

- **启动加载机制**：浏览器启动后，JS冷路径优先读取 IndexedDB 缓存的已进化能力，无需重复检测、重复修补，实现极速热启动。

### 6\.3 轻量化备选方案（极简场景）

针对轻量演示、低版本兼容、极简页面场景，可搭配二级缓存辅助优化，不替代 IndexedDB 核心存储：

- **内存临时缓存**：页面运行期间，将高频使用的短期记忆载入JS内存对象，提升读写速度，页面销毁自动清空，仅做运行时优化。

- **LocalStorage**：仅存储少量核心配置、版本标记，不存储大容量规范规则，规避其容量小、仅支持字符串的短板。

### 6\.4 最终落地结论

1. **纯浏览器网页不可用 LMDB**：受沙箱与权限限制，原生 LMDB 无法在网页环境运行，工业级桌面/服务端程序可继续沿用 LMDB。

2. **IndexedDB 为唯一最优解**：100%兼容 SEL\-TM 所有记忆持久化、自进化、冷热启动机制，功能零损耗、架构零改动、浏览器原生合规。

3. **架构统一对齐**：网页端 IndexedDB、桌面/服务端 LMDB，双层存储介质统一适配 SEL\-TM 七元组记忆机制，实现跨环境架构一致。

- **长期记忆库（𝕂）**：结构化机器可执行 W3C HTML/CSS 规范，永久固化于 LMDB，只读对齐官方标准

- **短期记忆集（𝕃）**：JS 冷路径运行时习得的布局能力、扩展状态、转移逻辑，动态写入 LMDB 持久化

- **运行机制**：浏览器启动时，LMDB 自动通过 mmap 映射热数据，无需 JS 手动回填缓存，实现无感极速热启动
